## 이 빌드의 목표

마일스톤 1 — **핵심 전투 루프 시연**

카드 기반 1:1 전투의 기본 흐름(드로우 → 카드 사용 → 적 턴 → 다음 턴)이 플레이 가능한 수준으로 동작하는 것을 목표로 합니다. 유물 시스템은 미포함이며 빌드 0.2.0으로 이월되었습니다.

## 구현된 기능

### 전투 기반 시스템

- **덱·드로우·버리기 더미**: 게임 시작 시 기본 덱에서 드로우. 손패가 소진되면 버리기 더미에서 덱을 재구성합니다. (#17)
- **손패 UI**: 손패 카드가 부채꼴 레이아웃으로 배치됩니다. (#17)
- **Draw / Discard Pile UI**: 덱과 버리기 더미의 잔량이 화면에 표시됩니다. (#51)
- **에너지 시스템**: 턴 시작 시 에너지가 회복됩니다. 코스트가 부족한 카드는 사용이 차단됩니다. (#43)
- **턴 종료 버튼**: 화면 버튼을 클릭해 플레이어 턴을 종료합니다. (#48)

### 카드 조작

- **마우스 드래그 타겟팅**: 카드를 드래그해 적에게 드롭하면 카드가 사용됩니다. 베지에 곡선 라인으로 타겟을 표시합니다. (#46)
- **카드 호버 확대**: 카드에 마우스를 올리면 카드가 확대 표시됩니다. (#46)
- **카드 애니메이션**: 드로우 시 더미에서 손패로, 사용 시 버리기 더미로 이동하는 애니메이션이 재생됩니다. (#18)

### 카드 효과

- **DealDamage**: 대상에게 피해를 입힙니다.
- **GainArmor**: 자신의 방어도를 올립니다.
- **DrawCard**: 덱에서 카드를 추가로 드로우합니다.
- **Weakness(취약)**: 대상에게 Weakness 상태를 부여하거나 강화합니다. Weakness 상태인 캐릭터는 공격 시 피해가 감소합니다.

### 적 시스템

- **적 스폰**: EnemySpawnTable에 등록된 적이 전투 시작 시 배치됩니다. (#19)
- **의도 표시**: 적이 다음 턴에 취할 행동(공격/방어 등)을 아이콘과 수치로 표시합니다. (#19)
- **적 턴 실행**: 플레이어 턴 종료 후 적이 EnemyActionPattern에 따라 행동합니다. (#19)

### 상태이상

- **Weakness**: 부여된 스택만큼의 턴 동안 받는 피해 증가 효과가 적용됩니다. 스택은 턴 종료 시 1 감소합니다. (#60, #61)

### 승패 판정

- 적의 체력이 0이 되면 **승리** 패널이 표시됩니다.
- 플레이어의 체력이 0이 되면 **패배** 패널이 표시됩니다.
- 패널 클릭 시 같은 씬이 리로드됩니다. (#23)

### 데이터 파이프라인

- **CardData ScriptableObject**: 카드 데이터를 에셋으로 관리합니다. (#16)
- **StringTableManager / CSVHelper**: 카드 설명 등 텍스트를 CSV에서 로드합니다. (#39)

## 플레이에 필요한 정보

### 조작 방법

| 조작 | 방법 |
|---|---|
| 카드 선택/사용 | 카드를 마우스로 드래그해 적 위에 드롭 |
| 턴 종료 | 화면 우측 턴 종료 버튼 클릭 |
| 카드 확인 | 카드에 마우스 호버 시 확대 표시 |

> **키보드 바인딩 없음** — 이 빌드는 마우스 전용입니다.

### 플레이 흐름

1. `BattleScene` 시작 → 손패 5장 드로우
2. 에너지 범위 내에서 카드를 적에게 드래그해 사용
3. 턴 종료 버튼 → 적 행동 → 플레이어 턴 시작 (에너지 회복, 드로우)
4. 적 처치 시 승리 패널, 플레이어 사망 시 패배 패널 → 클릭 시 씬 재시작

### 알려진 제한

- 보상·맵 이동 없이 전투 씬만 반복
- 유물 없음
- 덱 구성 변경 없음 (기본 덱 고정)
